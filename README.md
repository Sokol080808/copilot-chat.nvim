# copilot-chat.nvim

A simple Copilot-style chat plugin for Neovim.

## Important Notice

All code in this repository was generated by AI.

## What This Plugin Does

1. Opens a chat panel in Neovim.
2. Sends your prompts to the official GitHub Models API.
3. Streams model responses live into the chat.
4. Keeps multi-turn conversation history.
5. Edits code and previews file changes inline before applying them.

## Main Features

1. Official GitHub Copilot CLI integration (`npm install -g @github/copilot`)
2. Single chat window on the right side
3. Prompt input via popup (`vim.ui.input`)
4. Live token streaming via JSONL
5. Editing and inline source-buffer diff preview (not in chat)
6. Native Neovim highlight groups in preview: `DiffAdd`, `DiffDelete`, `DiffChange`

## Requirements

1. Neovim 0.8+
2. GitHub Copilot CLI installed globally: `npm install -g @github/copilot`
3. Authenticated Copilot session (`copilot login`)

## Commands

```vim
:CopilotChat        " Open chat window
:CopilotChatAsk     " Ask a question
:CopilotChatLogin   " Start GitHub login flow
:CopilotChatApply   " Apply pending previewed changes
:CopilotChatSkip    " Skip pending previewed changes
```

## Basic Usage

1. Run `:CopilotChat`.
2. Press `Enter` in the chat window (or run `:CopilotChatAsk`).
3. Type your prompt in `Copilot >`.

## Authentication Flow

Requires `copilot login` in advance.
If no `copilot` binary is found:

```bash
npm install -g @github/copilot
```

You can trigger login via the plugin's auth prompt sequence or run:

```vim
:CopilotChatLogin
```

## Configuration

```lua
require("copilot-chat").setup({
    system_prompt = "You are an AI programming assistant integrated into a Neovim editor.",
    enable_edit_requests = true,
    confirm_before_apply = true,
})
```

## Edit Preview And Apply Flow

When you ask Copilot to edit the active file:

1. The model gets the current file content.
2. The model returns a full updated file in one fenced code block.
3. The plugin shows a temporary inline preview in the source buffer.
4. Changed lines are highlighted with native diff groups.
5. You keep working in the file (scroll/edit/navigation is not blocked).
6. Run `:CopilotChatApply` or `:CopilotChatSkip` when ready.
7. `Apply` writes the final updated content.
8. `Skip` restores the original content.
